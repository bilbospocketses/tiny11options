## Iterative smoke passes

The matrix surfaced issues that were fixed in-cycle rather than deferred:

1. **Smoke pass 1** (`6d475d4`) — Step 1 columns were constrained to 720px by `section.form max-width`; light-mode contrast was too low; scratch placeholder text contradicted required-field gate.
2. **Smoke pass 2** (`18c12de`) — Path validation added (C# bridge + JS state + UI + tests). Two `PathValidationHandlers` for `validate-scratch` + `validate-output`. xUnit 124 → 136.
3. **Smoke pass 2 amend** (still `18c12de`'s replacement `b71124f`→`18c12de`) — Writability probe added to both validators. xUnit 136 → 138.
4. **Smoke pass 3** (`b3d4a2e`) — Left-column label spacing bumped 12 → 20px; deleted Log/Append hint paragraph.
5. **Smoke pass 4** (`2df8605`) — Reserved error slots (48px) so column doesn't jump when an error appears.
6. **Smoke pass 4b** (`e0b5277`) — Slot height 48 → 38px (48 was too generous).
7. **Smoke pass 5** (`46b7653`) — ISO source error moved from DOM textContent into `state.sourceError` so it survives renderStep; empty-text-node bug fixed (`|| null` instead of `|| ''`) so all three slots behave identically when empty.
8. **Smoke pass 6** (`82b239f`) — Label margin 20 → 14px; slot margin-top 4 → 2px (column was still too tall).
9. **Smoke pass 6b** (`53afc1f`) — Slot margin-top 2 → 6px (boxes were butted against the input row).
10. **Smoke pass 6c** (`b8e75b9`) — Slot min-height 38 → 30px (still some excess reserved space; 30 is enough for typical single-line errors).
